# Vivencias Azuis Design System

## Objetivo
Criar um design system enxuto e evolutivo para o Vivencias Azuis, com foco em:
- leitura longa em mobile
- consistencia visual entre blog e futuros produtos
- acessibilidade e confianca em temas sensiveis
- desempenho (Core Web Vitals)

## Escopo V1 (4-6 semanas)
- Foundations: cores, tipografia, spacing, radius, shadows, motion, tokens
- Componentes core: header + navegacao, card de artigo, pagina de artigo, CTA, blocos de destaque, formularios, controles de acessibilidade
- Guidelines: conteudo, acessibilidade, SEO, contribuicao

## Principios
1. Clareza acima de tudo: informacao direta, sem ruido visual
2. Calmo e acolhedor: paleta azul com neutros quentes
3. Acessivel por padrao: contraste e navegacao por teclado desde o inicio
4. Mobile-first real: leitura e acao rapida no celular
5. Performance e confianca: leveza, previsibilidade e transparencia

## Como usar
- Consulte foundations antes de criar novos componentes
- Prefira tokens semanticos (ex: color.text.primary)
- Documente cada componente com usage, a11y e tokens
- Evite variacoes nao planejadas

## Estrutura de pastas
- foundations/: cores, tipografia, spacing, tokens, motion
- components/: docs de cada componente core
- guidelines/: regras de uso, acessibilidade, conteudo e SEO
- patterns/: combinacoes de componentes (futuro)
- processes/: governanca e contribuicao

## Status
- V1: em construcao (foco em blog + paginas institucionais)
- V2: variacao high-contrast/dark e componentes para ferramentas interativas

## Roadmap curto
