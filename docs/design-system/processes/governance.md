# Governance

## Ownership
- Owner: dev/product
- Contribuidores: designers pontuais/freelancers

## Proposal
1. Abrir issue com contexto e problema
2. Incluir exemplos e referencias
3. Definir impacto e prioridade

## Review
- Design: consistencia com tokens e acessibilidade
- Code: performance e reuso
- Docs: uso e exemplos

## Implementation
- Componentes em React + TypeScript
- Tokens semanticos obrigatorios
- Documentacao em Markdown junto do componente

## Release
- Semver
- Changelog obrigatorio
- Comunicacao curta no blog interno

## Deprecation
- Aviso com 2 releases de antecedencia
- Guia de migracao simples
